# 🔮 Future Roadmap & TODOs

This document logs advanced mathematical modeling and algorithmic extensions planned for the Nest Egg Optimizer. 

Two primary methodologies have been proposed to transition the system from deterministic single-curve optimization to path-dependent stochastic models: **Stochastic Optimization** and **Reinforcement Learning**.

---

## 1. Stochastic Optimization (Markov Chain Monte Carlo)

### Current Limitations
The current Monte Carlo simulator stress-tests a **fixed deterministic strategy** (generated by the optimizer assuming a constant, average rate of return) against random annual volatility. It does not re-optimize or adjust the withdrawal allocation dynamically in response to market conditions within a given simulation path.

### Proposed Extension
Future versions will implement path-dependent **Stochastic Optimization** using Markov Chain Monte Carlo (MCMC) simulations:

- **Dynamic Recalculation**: Instead of applying a static withdrawal plan, the algorithm will dynamically adjust the withdrawal sequence at each simulated year based on the portfolio's actual current asset values and performance.
- **Path-Dependent Wealth Tax Rules**: The system will evaluate the probability distribution of portfolio survival under multiple pathways (e.g. sequence-of-returns risk) and solve for a dynamic policy $w(t, S_t)$ representing the optimal withdrawal vector at time $t$ given asset state vector $S_t$.
- **MCMC Sampling**: Use Metropolis-Hastings or Gibbs sampling to explore the parameter space of dynamic policies, finding the parameters that maximize the expected depletion age or minimize the probability of ruin.

---

## 2. Reinforcement Learning (Q-learning MDP Model)

An alternative approach is to model the retirement withdrawal problem as a **Markov Decision Process (MDP)** and solve it using Reinforcement Learning (RL).

```
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │                     MDP Environment                      │
  │  State (S_t): [TFSA, RRSP, Margin, Year, Tax Bracket]    │
  │  Action (A_t): [TFSA w/d %, RRSP w/d %, Margin w/d %]    │
  │  Reward (R_t): Longevity bonus + Tax minimization reward │
  │                                                          │
  └───────────────────────────▲──────────────────────────────┘
                              │
                    Action    │    State, Reward
                     (A_t)    │     (S_t, R_t)
                              │
  ┌───────────────────────────┴──────────────────────────────┐
  │                                                          │
  │                        RL Agent                          │
  │                  (Q-learning or DQN)                     │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
```

### MDP Formulation
- **State Space ($S_t$)**: A continuous vector containing:
  - Current TFSA account balance.
  - Current RRSP account balance.
  - Current Margin account principal and accrued capital gains.
  - Current year of retirement (age).
  - Active tax brackets and inflation rates.
- **Action Space ($A_t$)**: A continuous vector $[w_{tfsa}, w_{rrsp}, w_{marg}]$ representing the percentage of the spending target to draw from each account. The actions are subject to the constraint:
  $$w_{tfsa} + w_{rrsp} + w_{marg} = 1.0 \quad \text{and} \quad w_i \ge 0$$
- **Transition Function ($P(S_{t+1} | S_t, A_t)$)**: The financial burndown equations including random market growth (volatility), tax deductions, and inflation.
- **Reward Function ($R_t$)**: Designed to penalize portfolio depletion and reward tax efficiency:
  $$R_t = \begin{cases} 
    +1 & \text{if portfolio survives year } t \\
    -100 & \text{if portfolio depletes (ruin)} \\
    -\beta \cdot \text{Taxes Paid} & \text{to penalize unnecessary tax drag}
  \end{cases}$$

### RL Training Algorithms
- **Tabular Q-learning**: For discretized state spaces, learning the $Q(s, a)$ table indicating the value of each withdrawal mix.
- **Deep Q-Networks (DQN) / DDPG**: For continuous state and action spaces, using neural networks to approximate the optimal action-value function and policy.
- **Benefits**: Once trained, the RL policy network instantly outputs the optimal withdrawal action for any market scenario, naturally adapting to sequence-of-returns risk and bear/bull market cycles.
